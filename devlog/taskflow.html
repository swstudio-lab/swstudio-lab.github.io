실무 도구 반복 개선 작업물

장난감이 아니라 실무 도구를 만들고 싶었다

게임 다음에 작업물 탭을 채울 때, 흔한 투두리스트 예제 하나 만드는 건 의미가 없다고 생각했어요. 저는 행정직으로 일하고 있고, 실제 업무에서 투두리스트와 WBS(업무 분해 구조)를 따로따로 관리하는 게 불편했거든요. 그래서 이 둘을 하나로 합친, 진짜 업무에 쓸 수 있는 도구를 만들기로 했어요.

처음 버전 — 계층 구조가 핵심이었다

제일 먼저 만든 기능은 상위 항목 아래로 하위 항목을 무한히 쪼갤 수 있는 구조였어요. 1, 1.1, 1.1.2처럼 WBS 번호가 자동으로 붙고, 체크박스는 제일 아래 단계 항목에서만 누를 수 있게 했어요. 상위 항목은 그 밑에 딸린 하위 항목들이 몇 % 끝났는지로 진행률이 자동 계산되게 만들었고요. 실무에서 "이 프로젝트 지금 몇 % 진행됐어요?"라는 질문에 바로 답할 수 있어야 했거든요.

내보내기도 처음부터 세 가지로 만들었어요 — 보고용으로 텍스트 복사, 엑셀에서 열리는 CSV, 그리고 깔끔한 흑백 인쇄. 실제 업무 흐름을 그대로 반영한 선택이었어요.

막연한 아이디어를 구체화하기

어느 정도 완성되고 나서 "노션처럼 더 깔끔하고 편하게 쓸 수 있으면 좋겠다"는 생각이 들었어요. 근데 노션은 기능이 워낙 많아서, "노션처럼"이라는 말만으로는 저 스스로도 정확히 뭘 원하는 건지 애매했어요. 그래서 바로 손대지 않고 먼저 후보를 좁혀봤어요 — 더 깔끔한 디자인인지, 엔터키로 항목 추가인지, 드래그 정렬인지, 긴 메모 기능인지. 이렇게 선택지로 구체화하고 나니 진짜 필요한 부분만 골라서 반영할 수 있었어요.

💡 막연한 생각은 선택지로 좁혀서 정리한다 — "~처럼"이라는 감(感)은 저 스스로도 기준이 뚜렷하지 않을 때가 많아서, 바로 손대기보다 후보를 몇 가지로 나눠서 하나씩 확인하는 게 결과적으로 더 빨랐어요.

쓰면서 계속 늘어난 것들

  • 컬럼 확장 — 담당자만 있던 표에 우선순위·상태·시작일·마감일·진행률이 하나씩 추가되면서 테이블이 옆으로 길어졌어요. 그래서 필요 없는 컬럼은 숨길 수 있게 했고, 폰에서는 좌우로 스크롤되게 바꿨어요.
  • 예시 데이터 교체 — 처음엔 "출장 보고서 작성"이라는 예시로 시작했는데, 실제 업무 느낌이 안 나서 "프로젝트 기획서 작성"으로 바꿨어요. 진짜 쓸 도구는 예시부터 현실적이어야 한다고 생각했어요.
  • 이름 변경 — "할일 & WBS 관리"라는 기능 설명형 제목에서 "TaskFlow"라는 이름으로 바뀐 것도 이 시점이에요. 기능을 나열하던 도구에서, 정체성이 있는 하나의 제품으로 성격이 바뀐 순간이었어요.

어떤 도구로, 무엇을 했나

이 프로젝트는 AI(Claude)와 대화하면서 만들었어요. 직접 코드 문법을 하나하나 짠 건 아니지만, 그렇다고 그냥 던져놓고 결과만 받은 것도 아니에요. 제가 한 일과 AI가 한 일을 나누면 이래요.

  • 제가 한 것 — 어떤 컬럼이 필요한지, 예시 데이터가 실무와 맞는지, 이름을 뭘로 할지 같은 방향과 우선순위를 정했어요. 실제로 매번 써보면서 "이 부분은 불편하다", "이 색은 안 보인다" 같은 걸 캡처해서 하나씩 피드백했고요. 완성된 파일을 GitHub에 직접 올려서 실제로 배포하는 것도 제가 했어요.
  • AI가 한 것 — 제가 설명한 방향을 실제 HTML/CSS/JS 코드로 구현하고, 계층 구조 계산이나 CSV 인코딩 같은 세부 로직을 짰어요. 제가 놓친 부분(예: 컬럼이 많아지면 폰에서 어떻게 보일지)을 먼저 짚어주기도 했고요.

즉 "AI가 다 만들었다"도 아니고 "내가 다 짰다"도 아니에요. 기획과 판단은 제가, 구현은 AI와 같이 — 이 프로젝트 전체가 그런 방식으로 만들어졌어요.

지금 상태

지금은 여러 프로젝트를 전환하며 관리할 수 있고, 검색·정렬도 되고, 우선순위/상태는 색으로 바로 구분돼요. 여전히 손볼 곳은 있지만 (드래그 정렬, 엔터키 입력 같은 건 아직 미완성이에요), 실제로 제 업무에 써보면서 계속 다듬어가고 있는 도구예요.
